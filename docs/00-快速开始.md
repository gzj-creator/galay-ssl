# 00-快速开始

本文只覆盖三个最短闭环：

1. 构建仓库
2. 运行仓库内真实 include 示例
3. 安装后用 `find_package(galay-ssl REQUIRED)` 编译一个最小 consumer

## 前提条件

| 项目 | 要求 |
|------|------|
| 编译器 | 支持 C++23 |
| CMake | `>= 3.16`（默认构建）；import 示例需要 `>= 3.28` |
| OpenSSL | 可被 `find_package(OpenSSL REQUIRED)` 找到 |
| `galay-kernel` | 可被 `find_package(galay-kernel 4.0.0 REQUIRED CONFIG)` 找到 |
| 当前目录 | 建议始终从仓库根目录运行命令 |

## 直接构建

```bash
cmake -S . -B build \
  -DCMAKE_BUILD_TYPE=Release \
  -DENABLE_LTO=ON \
  -DCMAKE_PREFIX_PATH=/Users/gongzhijie/Desktop/projects/git/galay-kernel/_install-smoke-344
cmake --build build --parallel
```

这组命令对应“默认构建”：

- `BUILD_TESTING=OFF`（`BUILD_TESTS` 兼容别名也默认关闭）
- `BUILD_BENCHMARKS=ON`
- `BUILD_EXAMPLES=ON`
- `BUILD_MODULE_EXAMPLES=OFF`

常用开关：

- `-DDISABLE_IOURING=ON`：Linux 下强制回退 `epoll`
- `-DBUILD_MODULE_EXAMPLES=ON`：显式启用 import 示例；要求 CMake `>= 3.28`、生成器支持模块，并且编译器侧模块依赖扫描可用
- `-DBUILD_TESTING=ON`：显式启用 `test/` 与 `ctest`
- `-DBUILD_TESTING=OFF` / `-DBUILD_BENCHMARKS=OFF` / `-DBUILD_EXAMPLES=OFF`：按需裁剪

## 运行最小示例闭环

以下示例都来自 `examples/`，不是伪代码：

### 服务端

- 源文件：`examples/include/e1_echo.cc`
- Target：`e1_echo_include`
- 运行命令：

```bash
./build/bin/e1_echo_include 8443 certs/server.crt certs/server.key
```

### 客户端

- 源文件：`examples/include/e2_echo.cc`
- Target：`e2_echo_include`
- 不校验证书：

```bash
./build/bin/e2_echo_include 127.0.0.1 8443
```

- 使用 `certs/ca.crt` 校验证书：

```bash
./build/bin/e2_echo_include localhost 8443 certs/ca.crt
```

说明：带 CA 校验时建议使用 `localhost`，与仓库自带测试证书的主机名保持一致。

## 运行测试闭环

先显式打开测试：

```bash
cmake -S . -B build-test \
  -DCMAKE_BUILD_TYPE=Release \
  -DENABLE_LTO=ON \
  -DBUILD_TESTING=ON \
  -DBUILD_BENCHMARKS=OFF \
  -DBUILD_EXAMPLES=OFF \
  -DBUILD_MODULE_EXAMPLES=OFF
cmake --build build-test --parallel
ctest --test-dir build-test --output-on-failure
```

说明：

- `t1_socket` 主要覆盖 `SslContext`、`SslEngine`、`SslError`
- `t2_loopback` 才是仓库内现成的端到端回环 smoke test
- `t3_policy` 覆盖 ALPN、Session 复用、TLS 1.2 版本约束与 Cipher 策略

## 安装后最小 consumer

先安装到一个前缀：

```bash
cmake --install build --prefix "$PWD/.local"
```

最小 `CMakeLists.txt`：

```cmake
cmake_minimum_required(VERSION 3.16)
project(galay_ssl_consumer LANGUAGES CXX)

set(CMAKE_CXX_STANDARD 23)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

find_package(galay-ssl REQUIRED)

add_executable(consumer main.cpp)
target_link_libraries(consumer PRIVATE galay-ssl::galay-ssl)
```

最小 `main.cpp`：

```cpp
#include <galay-ssl/ssl/ssl_context.h>
#include <galay-ssl/async/ssl_socket.h>

int main() {
    galay::ssl::SslContext ctx(galay::ssl::SslMethod::TLS_Client);
    return ctx.isValid() ? 0 : 1;
}
```

附加说明：

- `galay-ssl-config.cmake` 会自动查找 `OpenSSL` 与 `galay-kernel 3.4.4+`
- 因此安装后环境里仍需要能找到 `galay-kernel`

## Modules 入口

仓库包含模块接口文件 `galay-ssl/module/galay_ssl.cppm`，但要注意：

- 默认构建不会生成 `galay-ssl-modules`
- 只有当 `BUILD_MODULE_EXAMPLES=ON`、CMake `>= 3.28`、生成器支持模块且编译器侧模块依赖扫描可用时，才会生成 `galay-ssl-modules`
- 若扫描能力缺失，根工程会给出 warning 并自动关闭 `BUILD_MODULE_EXAMPLES`
- 已实测：macOS `AppleClang 17` + CMake `4.0.2` + `Ninja` 在缺少扫描能力时不会生成 import 示例
- 已安装包当前不导出模块 target
- 对外稳定消费入口仍是 `galay-ssl::galay-ssl`
