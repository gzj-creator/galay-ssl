# 04-示例代码

本页只收录仓库内真实存在的示例与测试入口；不再混入未落地的说明性伪代码。

## 示例验证级别

- `已验证`：本仓库有真实源文件、真实 target，且适合直接运行
- `条件生成`：本仓库有真实源文件，但 target 只在特定工具链下生成
- `测试入口`：虽然不在 `examples/`，但能提供更强的端到端验证链路

## include 示例

### `e1_echo_include`

| 字段 | 内容 |
|------|------|
| 源文件 | `examples/include/e1_echo.cc` |
| Target | `e1_echo_include` |
| 验证级别 | 已验证 |
| 运行命令 | `./build/bin/e1_echo_include 8443 certs/server.crt certs/server.key` |
| 必需文件 | `certs/server.crt`、`certs/server.key` |
| 必需环境变量 | 无 |
| 说明 | 启动 TLS Echo 服务端 |

### `e2_echo_include`

| 字段 | 内容 |
|------|------|
| 源文件 | `examples/include/e2_echo.cc` |
| Target | `e2_echo_include` |
| 验证级别 | 已验证 |
| 运行命令 | `./build/bin/e2_echo_include localhost 8443 certs/ca.crt` |
| 必需文件 | `certs/ca.crt` 可选；不传则关闭证书校验 |
| 必需环境变量 | 无 |
| 说明 | 连接 Echo 服务端并执行一次请求/响应；带 CA 校验时建议使用 `localhost` |

## 用测试补足示例验证链路

某些用户关心的不只是“示例能启动”，而是“证书校验和回环是否真的跑通”。这时应优先看真实测试 target：

### `t2_loopback`

| 字段 | 内容 |
|------|------|
| 源文件 | `test/t2_loopback.cc` |
| Target | `t2_loopback` |
| 验证级别 | 测试入口 |
| 运行命令 | `./build/bin/t2_loopback` |
| 必需文件 | 构建阶段复制到 `build/bin/certs` 的测试证书 |
| 说明 | 启动本地 loopback server/client，客户端开启 CA 校验并验证 echo 回包 |

### `t1_socket`

| 字段 | 内容 |
|------|------|
| 源文件 | `test/t1_socket.cc` |
| Target | `t1_socket` |
| 验证级别 | 测试入口 |
| 运行命令 | `./build/bin/t1_socket` |
| 说明 | API 级回归，覆盖 `SslContext` / `SslEngine` / `SslError` 基础行为，以及 `loadCAPath()`、TLS 版本、Session cache/timeout 等策略 setter |

### `t3_policy`

| 字段 | 内容 |
|------|------|
| 源文件 | `test/t3_policy.cc` |
| Target | `t3_policy` |
| 验证级别 | 测试入口 |
| 运行命令 | `./build/bin/t3_policy` |
| 必需文件 | 构建阶段复制到 `build/bin/certs` 的测试证书 |
| 说明 | 两次 TLS 1.2 loopback 连接：验证 ALPN 协商、`setSession()` / `getSession()` / `isSessionReused()`、TLS 版本约束与 Cipher 策略 |

## 示例与能力覆盖映射

| 能力 | 真实入口 | 当前状态 |
|------|----------|----------|
| 基础 TLS Echo | `e1_echo_include` + `e2_echo_include` | 已有真实示例 |
| CA 验证 | `e2_echo_include`（传 `certs/ca.crt`） + `t2_loopback` | 已有真实入口 |
| 协程式建连/握手/收发/关闭 | `examples/include/*.cc`、`test/t2_loopback.cc` | 已有真实入口 |
| ALPN | `t3_policy` | 已有真实 test target |
| Session 复用 | `t3_policy` | 已有真实 test target |
| CA 路径 | `t1_socket` | 已有真实 test target |
| TLS 版本 / Cipher 策略 | `t1_socket` + `t3_policy` | 已有真实 test target |
| mTLS | 无独立示例 target | 仅证书资产存在 |
| 系统默认 CA | 无独立跨平台 smoke target | 运行环境依赖，不应假设所有平台等价 |
| 显式超时配置 | 无公开示例或 setter | 不应在文档中虚构 |

## 推荐阅读顺序

1. 先跑 `e1_echo_include` + `e2_echo_include`
2. 再跑 `t2_loopback`
3. 需要 API 细节时再查 [02-API参考](02-API参考.md)
