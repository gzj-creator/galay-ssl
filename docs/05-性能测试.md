# 05-性能测试

本页只描述仓库内真实存在的 benchmark 程序、命令和口径；没有附带新命令输出的数字一律不作为“当前最新性能结论”。

## 真实 benchmark target

| 源文件 | Target | 作用 |
|--------|--------|------|
| `benchmark/B1-ssl_bench_server.cc` | `B1-SslBenchServer` | Echo benchmark 服务端 |
| `benchmark/B1-ssl_bench_client.cc` | `B1-SslBenchClient` | Echo benchmark 客户端 |

## 构建前提

推荐构建：

```bash
cmake -S . -B build \
  -DCMAKE_BUILD_TYPE=Release \
  -DENABLE_LTO=ON
cmake --build build --parallel
```

说明：

- `scripts/S1-Bench.sh` 明确要求 `build/` 是 `Release + ENABLE_LTO=ON`
- 如果你只想做功能性 smoke，可以手动运行 benchmark 二进制；不一定要跑完整脚本预设

## 真实运行命令

### 服务端

```bash
./build/bin/B1-SslBenchServer 8443 certs/server.crt certs/server.key
./build/bin/B1-SslBenchServer 8443 certs/server.crt certs/server.key 4096 4
```

参数：

- `port`
- `cert_file`
- `key_file`
- `backlog`（可选，默认 `4096`）
- `worker_count`（可选，默认 `1`；大于 `1` 时启用 `SO_REUSEPORT + 多 scheduler + 多 listener`）

### 客户端

```bash
./build/bin/B1-SslBenchClient <host> <port> <connections> <requests_per_conn> [payload_bytes] [threads] [connect_retries]
```

示例：

```bash
./build/bin/B1-SslBenchClient 127.0.0.1 8443 200 500 47 4
./build/bin/B1-SslBenchClient 127.0.0.1 8443 10 200 65536 1
```

可选统计：

```bash
GALAY_SSL_STATS=1 ./build/bin/B1-SslBenchClient 127.0.0.1 8443 50 200 47 4
```

## `scripts/S1-Bench.sh` 的真实预设

脚本会顺序运行这些场景：

- `1 x 100` 预热
- `1 x 1000`
- `1 x 5000`
- `1 x 10000`
- `10 x 1000`
- `50 x 200`
- `100 x 100`

这里的 `payload_bytes`、`threads`、`connect_retries` 使用 benchmark 客户端默认值：

- `payload_bytes = 47`
- `threads = 1`
- `connect_retries = 3`

## Cross-Language Remote Benchmark

除了本地 `S1-Bench.sh`，仓库现在还有一套跨语言远端入口：

- `scripts/S2-CrossLangBench.sh`
- 本地 wrapper，负责同步代码、启动远端 runner、回传结果
- `scripts/S2-CrossLangBench.remote.sh`
- 远端 Linux runner，负责构建、绑核、probe、warmup、formal runs、CSV 汇总

典型命令：

```bash
bash scripts/S2-CrossLangBench.sh \
  --remote-host 140.143.142.251 \
  --remote-user ubuntu \
  --remote-password '***' \
  --remote-root /home/ubuntu/tmp/galay-ssl-crosslang-bench \
  --server-cpus 0-1 \
  --client-cpus 2-3 \
  --streaming-connections 64 \
  --streaming-requests 500 \
  --streaming-payload 256 \
  --handshake-connections 128 \
  --handshake-requests 1 \
  --handshake-payload 47 \
  --galay-ssl-workers 4 \
  --build-jobs 1 \
  --threads 4 \
  --warmup-runs 1 \
  --formal-runs 5
```

如果远端机器还没有 `cargo` / `go`，可以附加：

```bash
--bootstrap-toolchains
```

说明：

- `--galay-ssl-workers` 只作用于 `galay-ssl` 的 `B1-SslBenchServer`
- 它用于验证 `handshake-heavy` 场景里 benchmark 服务端并发模型对结果的影响
- 如果目标是做更公平的 cross-language 对比，建议把它设到接近 `--server-cpus` 的核数
- `--build-jobs` 会限制远端 CMake / Cargo / Go 构建并行度；远端机器规格较小时建议显式设成 `1`

## CPU 保护口径

`S2-CrossLangBench.remote.sh` 默认会对每个场景先做 probe，再决定正式负载：

- benchmark 服务端和客户端使用互不重叠 CPU 集
- 默认阈值：
- 组合平均 CPU 不超过主机总容量的 `70%`
- 单侧平均 CPU 不超过其绑定核容量的 `80%`
- 如果任一实现超过阈值，runner 会统一降低该场景的连接数，再重新 probe

因此最终 formal runs 可能不会使用你最初传入的连接数，而是使用 probe 后的安全连接数。结果目录里会把最终采用的连接数一并记录下来。

## 输出指标

`B1-SslBenchClient` 当前会输出：

- `Connections`
- `Requests per connection`
- `Payload bytes`
- `Threads`
- `Total requests`
- `Total errors`
- `Error breakdown`（仅当 `Total errors > 0`）
- `Total bytes sent`
- `Total bytes received`
- `Duration`
- `Requests/sec`
- `Throughput`
- `SSL IO Stats`（仅当设置 `GALAY_SSL_STATS=1`）
- `Avg recv chunk bytes`（仅当设置 `GALAY_SSL_STATS=1` 且有接收块）

当前 benchmark **不会**输出 `Avg latency`、`P50 / P95 / P99 latency` 之类的延迟分位数指标；如果需要这类口径，必须先修改 benchmark 程序本身。

## 方法学边界

当前客户端实现还有两个必须一并披露的前提：

- benchmark 客户端会固定加载 `certs/ca.crt`
- 但随后会把验证模式硬编码为 `SslVerifyMode::None`

因此现有结果更接近“TLS 握手 + 加密 echo 吞吐 / QPS”测量，而**不包含**对端证书校验成本。解释结果时不能把它当成“开启证书校验后的端到端真实性能”。

## 推荐验证方式

如果你想确认 benchmark 入口是好的，而不是立刻追求完整性能结论，建议：

1. 启动 `B1-SslBenchServer`
2. 先运行一个极小 `B1-SslBenchClient` smoke
3. 确认 `Total errors: 0`
4. 再决定是否运行 `scripts/S1-Bench.sh`
