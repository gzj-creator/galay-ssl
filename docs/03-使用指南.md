# 03-使用指南

本页说明仓库目录、构建开关、脚本行为和直接运行方式。

## 目录结构

| 路径 | 说明 |
|------|------|
| `galay-ssl/` | 库源码与公开头 |
| `examples/` | 真实示例 target |
| `test/` | 真实测试 target |
| `benchmark/` | 真实 benchmark target |
| `scripts/` | 仓库内提供的构建/测试/benchmark 脚本 |
| `certs/` | 示例和手动 benchmark 可直接使用的证书 |
| `test/certs/` | 测试证书与生成脚本 |

## CMake 选项

根工程当前使用的主要选项如下：

| 选项 | 默认值 | 作用 |
|------|--------|------|
| `BUILD_TESTS` | `ON` | 构建 `T1-SslSocketTest`、`T2-SslLoopbackSmoke`、`T3-SslAdvancedSmoke` |
| `BUILD_BENCHMARKS` | `ON` | 构建 `B1-SslBenchServer`、`B1-SslBenchClient` |
| `BUILD_EXAMPLES` | `ON` | 构建 include 示例与可能的 import 示例 |
| `BUILD_SHARED_LIBS` | `ON` | 构建共享库 |
| `DISABLE_IOURING` | `OFF` | Linux 下关闭 `io_uring`，改用 `epoll` |
| `ENABLE_LTO` | `ON` | Release/RelWithDebInfo 启用 IPO/LTO |
| `BUILD_MODULE_EXAMPLES` | `OFF` | 仅在显式开启后，且模块工具链支持时生效 |

## 官方脚本的真实行为

### `./scripts/run.sh build`

- 配置目录：`build/`
- 使用 `cmake -S/-B` + `cmake --build`，不再写死 `make`
- 显式启用：`BUILD_TESTS=ON`、`BUILD_BENCHMARKS=ON`
- 通过默认值构建示例
- 显式传 `BUILD_MODULE_EXAMPLES=OFF`，因此不会生成 import 示例
- 使用 `Release + ENABLE_LTO=ON`
- 可在 `build` 子命令后追加 CMake 配置参数，例如 `./scripts/run.sh build -G Ninja`

### `./scripts/run.sh test`

- 顺序运行 `build/bin/T1-SslSocketTest`
- 顺序运行 `build/bin/T2-SslLoopbackSmoke`
- 顺序运行 `build/bin/T3-SslAdvancedSmoke`

### `./scripts/run.sh bench`

- 调用 `scripts/S1-Bench.sh`
- 仅适用于 `build/` 且要求 `Release + ENABLE_LTO=ON`
- 会启动 `B1-SslBenchServer`
- 会对 `B1-SslBenchClient` 使用固定若干组连接数/请求数组合

### `./scripts/check.sh`

- 重新执行 `T1-SslSocketTest`
- 重新执行 `T2-SslLoopbackSmoke`
- 重新执行 `T3-SslAdvancedSmoke`
- 检查 benchmark 二进制是否存在
- 检查 `build/bin/certs` 内测试证书是否复制到位
- 不执行真实 benchmark 压测

## 手动运行命令

### 测试

```bash
./build/bin/T1-SslSocketTest
./build/bin/T2-SslLoopbackSmoke
./build/bin/T3-SslAdvancedSmoke
```

### include 示例

```bash
./build/bin/E1-SslEchoServer-Include 8443 certs/server.crt certs/server.key
./build/bin/E2-SslClient-Include 127.0.0.1 8443
./build/bin/E2-SslClient-Include localhost 8443 certs/ca.crt
```

### import 示例

仅当下列条件同时成立时 target 才存在：

- `BUILD_MODULE_EXAMPLES=ON`
- CMake `>= 3.28`
- 生成器为 `Ninja` 或 `Visual Studio`
- 编译器侧模块依赖扫描可用（例如 `CMAKE_CXX_SCANDEP_SOURCE` 非空）
- 根工程创建了 `galay-ssl-modules`

对应命令：

```bash
./build/bin/E1-SslEchoServer-Import 8443 certs/server.crt certs/server.key
./build/bin/E2-SslClient-Import localhost 8443 certs/ca.crt
```

### benchmark

```bash
./build/bin/B1-SslBenchServer 8443 certs/server.crt certs/server.key
./build/bin/B1-SslBenchClient 127.0.0.1 8443 200 500 47 4
```

可选开启 benchmark 侧 SSL IO 统计：

```bash
GALAY_SSL_STATS=1 ./build/bin/B1-SslBenchClient 127.0.0.1 8443 50 200 47 4
```

## 工作目录要求

为了让相对路径稳定，建议全部从仓库根目录运行：

- 示例和 benchmark 使用根目录下的 `certs/`
- `scripts/check.sh` 与 `scripts/S1-Bench.sh` 会切换到 `build/bin`，因此它们依赖构建阶段复制进去的 `build/bin/certs`

## 建议验证顺序

如果你想最短验证“库可用”，建议按这个顺序：

1. `./scripts/run.sh build`
2. `./scripts/run.sh test`
3. `./build/bin/T2-SslLoopbackSmoke`
4. `./build/bin/T3-SslAdvancedSmoke`
5. `./build/bin/E1-SslEchoServer-Include ...` 与 `./build/bin/E2-SslClient-Include ...`
5. `./build/bin/B1-SslBenchServer ...` 与一个轻量 `B1-SslBenchClient` smoke
